# 👑 Documentación de Administrador

## 1. Gestión de Versiones
El administrador es responsable de supervisar las ramas de Git y asegurar que la versión en `main` sea siempre la versión estable.

## 2. Control de Acceso (Dashboard)
Actualmente, no existe un panel de administración centralizado. Los cambios en el contenido (como el versículo por defecto) se realizan directamente en `src/context/JournalContext.tsx`.

## 3. Gestión de Datos Agregados
El administrador no tiene acceso a los datos de los usuarios, preservando la privacidad total del sistema descentralizado.
